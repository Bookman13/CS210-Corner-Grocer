# CS 210 Portfolio Reflection – Corner Grocer Project

## Project Summary

The Corner Grocer project was designed to analyze grocery purchase data and track the frequency of items purchased throughout the day. The program reads data from an input file, stores item frequencies using a map, creates a backup data file, and provides a menu driven interface that allows users to search for item frequencies, display all item frequencies, and view a histogram representation of purchase counts. The goal of the project was to help the Corner Grocer better understand customer purchasing habits and organize its produce section more effectively.

## What I Did Particularly Well

One aspect of the project that I completed well was implementing a clear object oriented design. I created a dedicated CornerGrocer class to manage data processing and reporting functions. Separating the program into multiple source files improved organization and readability. I also successfully implemented file input and output, data storage using maps, and menu-driven user interaction.

## Areas for Improvement

One area that could be improved is the search functionality. Currently, the program handles basic case sensitivity by capitalizing the first letter of the user's input. A more robust solution would allow fully case insensitive searches regardless of how the user enters the item name. Another possible improvement would be sorting items by frequency rather than alphabetically. These enhancements would make the application more user-friendly and efficient.

## Challenges and How I Overcame Them

The most challenging part of the project was working with maps and file processing. Initially, I needed to better understand how to store item frequencies and update values as items were read from the input file. I overcame these challenges by reviewing course materials, practicing with smaller examples, and testing my code incrementally. Resources such as ZyBooks, course examples, Visual Studio debugging tools, and instructor guidance helped strengthen my understanding.

## Transferable Skills

This project strengthened several skills that will transfer directly to future coursework and software development projects. These skills include object-oriented programming, file input and output, use of data structures such as maps, debugging, program testing, and user input validation. These concepts are fundamental to many software engineering and computer science applications.

## Maintainability, Readability, and Adaptability

I made the program maintainable by separating functionality into different files and organizing related tasks into class methods. Meaningful variable and function names were used throughout the project, and comments were included to explain important sections of code. This structure makes the program easier to read, modify, and expand in the future. Additional features, such as enhanced searching or graphical output, could be added without significantly changing the overall design.
